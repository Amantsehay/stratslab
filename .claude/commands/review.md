# Code Review

Perform a comprehensive code review of the provided code or files following best practices, design patterns, and project conventions.

## Instructions

- Review code for quality, maintainability, and adherence to best practices
- Check for common issues: naming, complexity, duplication, error handling
- Verify tests are adequate and cover edge cases
- Ensure design follows project patterns and conventions
- Look for security vulnerabilities and performance issues
- Provide constructive feedback with specific, actionable recommendations
- Highlight what's working well and what could be improved
- Consider maintainability, extensibility, and testability

## Review Scope

Focus on the following aspects:
1. **Code Quality** - Readability, naming, structure, complexity
2. **Testing** - Coverage, edge cases, test quality and clarity
3. **Performance** - Efficiency, bottlenecks, resource usage
4. **Security** - Input validation, error handling, vulnerability prevention
5. **Design** - Patterns, architecture, adherence to conventions
6. **Documentation** - Comments, docstrings, clarity of intent
7. **Maintainability** - Future changes, extensibility, clarity

## Relevant Files

Reference these files when reviewing:
- `README.md` - Project overview and conventions
- `stratslabapi/**` - Backend API patterns and conventions
- `tests/**` - Testing patterns and standards

## Review Output

Provide a structured review including:
- **Summary** - Overall assessment of the code quality
- **Strengths** - What's working well and why
- **Improvements** - Specific areas for enhancement with reasoning
- **Security Concerns** - Any security issues found and fixes
- **Performance Notes** - Performance considerations and optimizations
- **Testing Feedback** - Coverage gaps and test quality issues
- **Recommendations** - Specific, actionable next steps
